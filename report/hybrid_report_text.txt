\subsection{CNN-LSTM hybrid model}

For our part of the project, we implemented a hybrid CNN-LSTM architecture for multivariate time-series forecasting on the Jena Climate dataset. The goal was to predict the target variable $T$ (degC) from historical meteorological measurements.

The shared preprocessing pipeline first kept only numeric columns and converted the original 10-minute sampling into hourly sampling by selecting every 6th row. After preprocessing, the dataset had shape $(70092, 14)$. The data were then split chronologically into training, validation, and test subsets in order to avoid data leakage. All inputs and the target variable were standardised using the shared pipeline.

The hybrid model combines two different ideas. First, a 1D convolutional block extracts local temporal patterns from the input sequence. Then, an LSTM block models longer sequential dependencies using these CNN-extracted features. In this way, the hybrid architecture tries to combine the local pattern extraction ability of CNN with the sequence modelling ability of LSTM.

The model receives input of shape $(\text{batch}, \text{window\_size}, 14)$ and outputs a single predicted temperature value. Inside the model, the input is first permuted for Conv1D processing, passed through convolutional layers with ReLU activations and MaxPool1d, then permuted back and processed by a multi-layer LSTM. The hidden state of the last time step is passed through a linear layer to produce the final scalar prediction.

We used several parameter choices specific to the hybrid design:
\begin{itemize}
    \item \textbf{Input window size: $48$.} Because the hybrid contains a MaxPool1d layer that reduces the temporal dimension, a larger input window was chosen to preserve sufficient sequential context for the LSTM part.
    \item \textbf{CNN channels: $64$.} This gives the convolutional part enough capacity to extract informative local features before passing them to the recurrent block.
    \item \textbf{Kernel size: $5$.} A wider temporal kernel allows the model to capture richer short-term temporal patterns before sequential modelling.
    \item \textbf{LSTM hidden size: $32$.} Since the CNN already compresses the original 14-dimensional input into higher-level features, a smaller hidden size is sufficient and helps keep the model compact.
\end{itemize}

Other training parameters were kept the same as in the shared project pipeline:
\begin{itemize}
    \item optimizer: Adam,
    \item loss function: MSELoss,
    \item learning rate: $10^{-3}$,
    \item batch size: $64$,
    \item maximum epochs: $50$,
    \item early stopping patience: $10$,
    \item scheduler: ReduceLROnPlateau,
    \item random seed: $42$.
\end{itemize}

\paragraph{Tested configurations.}
We evaluated two hybrid configurations that differed only in the number of LSTM layers:

\begin{itemize}
    \item Config A: $2$ LSTM layers
    \item Config B: $3$ LSTM layers
\end{itemize}

This was a clean ablation of recurrent depth inside the hybrid setting. The motivation was to test whether the LSTM benefits from additional depth after the CNN has already extracted local features.

\paragraph{Results.}
The final shared evaluation on the Jena Climate test set produced the following results:

\begin{table}[h]
\centering
\begin{tabular}{lcccc}
\hline
Model & Window & Params & MAE [$^\circ$C] & RMSE [$^\circ$C] \\
\hline
Hybrid Config A ($2$ layers) & $48$ & $46{,}113$ & $0.4701$ & $0.6636$ \\
Hybrid Config B ($3$ layers) & $48$ & $54{,}561$ & $0.5546$ & $0.7464$ \\
\hline
\end{tabular}
\caption{Final hybrid CNN-LSTM results on the test set}
\end{table}

The results show that Config A outperformed Config B on both evaluation metrics. Hybrid Config A achieved lower MAE and lower RMSE while also using fewer parameters. In contrast, adding a third LSTM layer increased model size without improving performance.

More specifically:
\begin{itemize}
    \item Hybrid Config A achieved test MAE of $0.4701\,^\circ$C and RMSE of $0.6636\,^\circ$C.
    \item Hybrid Config B achieved test MAE of $0.5546\,^\circ$C and RMSE of $0.7464\,^\circ$C.
\end{itemize}

This suggests that in the hybrid setting, increasing recurrent depth beyond two LSTM layers was not beneficial. Once the CNN block extracted useful local features, a deeper recurrent block likely introduced unnecessary complexity rather than better generalisation.

Another important point is that Hybrid Config A performed very strongly in the overall project comparison. It was one of the best-performing models in the project and represented a very good trade-off between predictive accuracy and model complexity. Although the strongest LSTM achieved slightly better absolute accuracy in the final ranking, the hybrid model remained highly competitive while using substantially fewer parameters.

Therefore, for the hybrid part of the project, the final selected model is \textbf{Hybrid Config A} with $2$ LSTM layers, because it achieved the best overall hybrid performance on the Jena Climate test set.
